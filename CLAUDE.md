# korobki — параметрическая коробка TORN для лазерной резки

Разговоры по проекту ведём на русском.

## Что это

Витринная коробка-дисплей из фанеры со сдвижной крышкой из оргстекла:

- **Верхняя деталь** — рамка со сквозным **окном** (скруглённый квадрат). Пазы на
  3 краях, четвёртый край ровный — там щель.
- **Оргстекло** (панель с полосой и надписью TORN, отдельный материал ~2.1 мм) —
  сдвижная крышка: входит в щель между верхом передней стенки и верхней деталью,
  ездит в канале высотой `gap` под окном.
- **Две вкладки** — вклеиваются вдоль левой/правой стенок, их верхние торцы +
  верх передней стенки образуют плоскость, по которой ездит крышка.
- **Дно** — пластина с шипами насквозь через стенки (заподлицо снаружи).
- **Стенки**: задняя и боковые шипами входят в пазы верхней детали; передняя ниже
  на `gap` (щель). Между собой — малые шипы в вырезы боковых.

## Файлы

- `korobka.cdr` — исходник CorelDRAW, не трогаем.
- `korobka.svg` — **эталон** (очищенная конверсия CDR: раскладки 3 и 3,4 мм + панель TORN).
- `box-geometry.js` — **единственный источник правды по геометрии** (UMD: браузер + node).
- `index.html` — веб-конфигуратор: параметры → 3D-превью (SVG, перспектива,
  вращение мышью, «разнести», ход крышки) → скачивание SVG-раскладок.
  Открывается двойным кликом, без сервера и зависимостей.
  Фокус/наведение на параметр рисует чертёжную размерную линию (двусторонние
  стрелки, выносные, значение) и подсвечивает затронутые детали акцентом;
  короткие размеры — стрелки снаружи (dimsFor/drawDim/highlightFor).
- `verify.js` — повершинная сверка движка с эталоном: `node verify.js`.
- `generate_box.py` — **legacy** CLI-генератор старой модели (только толщина,
  без новых параметров). Актуальный экспорт — из index.html.

## Параметры (все в мм)

| параметр | дефолт | смысл |
|---|---|---|
| W, H | 99.4 × 99.4 | внутренние размеры полости по дну (ядро детали «дно») |
| D | 44.2 | полезная глубина ПОД крышкой; полная полость Dfull = D + acr + tech |
| t | 3.0 | толщина фанеры |
| acr | 2.1 | толщина оргстекла |
| gap | 2.2 | щель/канал крышки (в эталоне 2.2, не 2.3!) |
| tech | 0.3 | техзазор глубины над содержимым |
| ov | 0.4 | натяг шип/паз суммарно (по 0.2/сторона) — компенсация реза |
| frame | 4.2 | рамка окна от внутреннего края |
| lip | 0.6 | выступ крышки наружу за переднюю стенку |
| tornOff | 6.5 | отступ полосы с надписью TORN от переднего края крышки |

## Модель (реверс-инжиниринг эталона, сравнение раскладок 3.0/3.4)

- Каждая координата линейна по t: `a + b·t`, b ∈ {0,1,2}.
- Габарит детали = внутренний размер + 2t; глубина шипов/пазов = ровно t.
- **Стыки пропорциональны**: большой номинал = сторона × 52/99.4,
  малый = Dfull × 14/46.6; центр малых стыков = Dfull × 21.5/46.6 от дна.
  Шип = номинал + ov/2, паз = номинал − ov/2.
- Паз-приёмник шипа дна на задней стенке в эталоне без натяга и стенка на 0.4
  длиннее боковых (аномалия ручной подгонки) — **в движке нормализовано**
  (verify.js допускает diff 0.4 на задней).
- Окно = (W−2·frame)×(H−2·frame), r=1.6, центр = центр полости (отступ frame+t от края детали).
- Оргстекло: ширина W−0.6 (по 0.3 зазор), длина H+t+lip, r=1.3891;
  полоса высотой 22.0232 в `tornOff` (эталон 6.5) от переднего края; буквы
  TORN — путь из CDR, вшит строкой (координаты pt, группа scale(25.4/72)),
  якорь к центру панели и верху полосы. Буквы 12.47 мм высотой, начинаются
  в 4.67 мм от переднего края полосы — отсюда предупреждение «уходит под
  рамку» при tornOff < t+lip+frame−4.67.
- Вкладки: W × (Dfull − gap).

## Формат экспорта

- SVG в мм 1:1 (`width="..mm"`, viewBox мм), hairline 0.0762 мм (=0.216 pt эталона).
- Цвета как в эталоне: рез `#1b1918`, верхняя деталь и окно `#000000`.
- Два файла: `…_fanera.svg` (8 деталей) и `…_akril.svg` (панель+полоса+TORN) —
  разный материал, режется отдельно.
- Подпись: `3ММ — 99,4×99,4×44,2` (запятые, кириллица, как в эталоне).

## Верификация после правок геометрии

`node verify.js` — сверяет движок с korobka.svg. Ожидания: все детали 0.000;
задняя ≤ 0.4 (нормализация аномалии); окно 91×91 при дефолтах. Экспорт из
страницы проверялся отдельно тем же способом — при правках exportSVG повторить
(вынуть SVG через подмену URL.createObjectURL в browser_evaluate).

## Открытые вопросы

- Полоса 98.8×22.0232 на оргстекле (в 6.5 мм от переднего края) и надпись TORN:
  физический смысл не подтверждён пользователем (гравировка? матовая зона под
  хват?). Текущее поведение при ресайзе: полоса тянется на всю ширину, высота
  фиксированная, привязка к переднему краю, надпись фикс. размера по центру.

## Готчи

- 1 pt = 25.4/72 мм; в эталоне всё круглое в мм (8.5039 pt = 3.0, 9.6378 pt = 3.4,
  281.7638 pt = 99.4).
- Hairline-линии не видны в мелком масштабе: для превью временно
  `sed 's/stroke-width:0.0762/stroke-width:0.5/'` + Inkscape → PNG.
- CDR→SVG: `/Applications/Inkscape.app/Contents/MacOS/inkscape file.cdr
  --export-area-drawing --export-plain-svg --export-type=svg --export-filename=out.svg`.
- Inkscape `--query-all` даёт bbox в px (96/дюйм) — третья система единиц, не путать.
- Playwright MCP не открывает `file://` — для теста страницы поднять
  `python3 -m http.server`; в обычном браузере file:// работает.
- В 3D-рендере страницы базис камеры: `right = up × fwd` (не наоборот!) — иначе
  сцена зеркалится (ловили на надписи TORN).
- Figma не подходит для правок раскладок: только px, теряет мм.
